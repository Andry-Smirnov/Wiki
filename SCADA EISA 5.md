# SCADA EISA 5

## Введение

SCADA EISA 5 предназначена для создания верхнего уровня автоматизированных систем (подсистем) управления технологическими процессами (АСУ ТП). EISA является системой, которая работает в реальном времени и позволяет оператору, используя входящие в систему аппаратные и программные средства, обеспечивать автоматизированное управление, надежную и экономичную работу технологического оборудования.

Система представляет собой два или более компьютера, соединенных между собой по локальной сети. Эта же сеть может быть использована для связи SCADA EISA с другой АСУ ТП объекта. Основным видом предоставления информации оперативному персоналу являются мнемосхемы, информация на которых меняется в реальном времени. Дистанционное управление оборудованием ведется оперативным персоналом с функциональной клавиатуры. При этом оператор видит на мнемосхемах оперативные изменения информации. Аварийные и предупредительные сообщения, возникающие в системе, приводят к выводу поверх любой мнемосхемы окна текущих сообщений с одновременной фиксацией в архиве. Аналоговые величины хранятся в системе в виде графиков.

Существует два режима просмотра графиков: история изменения параметра за определенное время; слежение за параметром в реальном времени (аналог самописца).

## Структура каталогов SCADA EISA 5

В стандартной конфигурации в корневом каталоге SCADA EISA 5 присутствуют следующие подкаталоги:

- `archive` - каталог, в котором сохраняются суточные архивы аналоговых параметров и содержится файл `archive.dat`, в котором сохраняются все информационные сообщения, фиксируемые EISA,
- `backup` - каталог, в котором хранятся созданные бекапы,
- `base` - каталог баз данных настроек,
- `document` - содержит файлы документации,
- `image` - содержит файлы изображений элементов мнемосхемы, отсортированных в каталоги,
- `mnemobase` - содержит файлы мнемосхем с расширениями: `.dbf`, `.lgm` и `.mdx`,
- `option` - содержит файлы настроек SCADA EISA 5,
- `program` - содержит дополнительные системные инструменты и инструменты разработчика,
- `resource` - содержит файлы шрифтов, курсоров и звуков, используемых в SCADA EISA 5,
- `source` - содержит файлы исходного кода SCADA EISA 5,
- `source-test` - содержит файлы исходного кода тестов SCADA EISA 5,
- `update` - содержит файлы обновлений.

Файлы, находящиеся в корневом каталоге SCADA EISA 5:

- `backup.cmd` и `backup.sh` - скрипты создания бекапа текущей конфигурации SCADA EISA 5;
- `clean-lib.cmd` и `clean-lib.sh` - скрипты очищающие подкаталог `source\lib`;
- `clean-sources.cmd` и `clean-sources.sh` -  скрипты очищающие подкаталоги `source` и `source-test`: удаляет подкаталог `lib`, все подкаталоги `backup`, файлы с расширением `.~*`, `.bak`, `.ddp`, `.dcu` и `.cbk`;
- `libopenal.dylib`, `libopenal.so`, `libsndfile.dylib`, `libsndfile.so`, `sndfile.dll` и `soft_oal.dll` - файлы библиотеки OpenAL, необходимы для вывода звука в Linux (`so`), MacOS (`dylib`) и Windows (`dll`);
- `README.md` - текущий файл - краткое описание репозитория SCADA EISA 5;
- `scadaeisal` или `scadaeisal.exe` - исполняемый файл SCADA EISA 5;
- `scadaeisal.log` - лог-файл с информацией по работе SCADA EISA 5.

Скрипты `.cmd` исполняются под ОС Windows, скрипты `.sh` под ОС семейства *nix.
