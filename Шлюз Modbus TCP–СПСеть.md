# Шлюз Modbus TCP–СПСеть

Предварительные идеи по реализации шлюза Modbus TCP <=> СПСеть

## 3.5 Функция `0x1D`/`0x03`(стр. 12¹) – чтение параметров

Реализуется посредством Modbus-функции `0x17 Read/Write Multiple registers` (стр. 38²) и служит только для чтения числовых значений

Запрос

| Поле | Длина | Значение |
| --- | --- | --- |
| Код функции | 1 байт | `0x17` – Чтение/запись регистров хранения |
| Начальный адрес чтения | 2 байта³ | Пока неопределено, поэтому `0` |
| Количество регистров чтения | 2 байта³ | 2…124 – зависит от числа запрошенных параметров и их формата, по 2 регистра на параметр |
| Начальный адрес записи | 2 байта³ | `0` (в планах использовать в качестве базового номера канала) |
| Количество записываемых регистров | 2 байта³ | Соответствует количеству запрашиваемых параметров, поскольку отклик не более 124 регистров – не более 62 |
| Количество записываемых байт | 1 байт | Удвоенное значение предыдущего параметра |
| Данные | – | Содержат запрашиваемые 16-разрядные указатели (= канал × `1000`+ параметр) |

Отклик при отсутствии ошибок в запросе и успешном выполнении операции

| Поле | Длина | Значение |
| --- | --- | --- |
| Код функции | 1 байт | `0x17` |
| Байт в отклике | 1 байт | Соответствует длине отклика, всегда кратно 4 |
| Данные | – | Содержат структуры, соответствующие запрошенным указателям, в порядке их перечисления в запросе |

Формат данных в отклике зависит от полученного по интерфейсу СПСеть ответа и может содержать 2 или 4 регистра на каждый запрошенный параметр, а в случае наличия в запросе некорректного указателя (см. 3.4 Указатели и информация при символьном обмене, стр.11¹) в отклике вернётся exception `02 ILLEGAL DATA ADDRESS` (см. 7 MODBUS Exception Responses, стр. 48²). Первые два регистра структуры всегда содержат значение запрошенного параметра и содержат:

- 32-разрядный float в формате IEEE-754, если в отклике получена строка, содержащая точку и соответствующая числовому значению
- 32-битное целое без знака, если строка содержит только цифры
- 32-битное целое со знаком в дополнительном коде, если первый символ строки – знак минус (или плюс – но вроде в откликах такой знак не предусмотрен)

Следующие два регистра могут отсутствовать, а при наличии содержат метку времени – наличие зависит от конкретного параметра конкретного прибора – либо всегда есть, либо никогда нет.

### Чтение ASCII-параметров

Реализуется посредством Modbus-функции `0x01 Read Coils` для чтения одного ASCII-значения.

Запрос

| Поле | Длина | Значение |
| --- | --- | --- |
| Код функции | 1 байт | `0x01` – Чтение катушек |
| Начальный адрес | 2 байта³ | Указатель (= канал × `1000`+ параметр) |
| Количество катушек | 2 байта³ | Количество читаемых байт × 8 |
| Количество читаемых байт | 1 байт | Определяется записываемым указателем |
| Данные | – | ASCII-строка для записи |

Отклик при отсутствии ошибок в запросе и успешном выполнении операции

| Поле | Длина | Значение |
| --- | --- | --- |
| Код функции | 1 байт | `0x01` |
| Байт в отклике | 1 байт | Соответствует длине отклика |
| Данные | – | Содержит запрошенный ASCII-параметр, если запрошенных байт больше длины строки – последние байты заполняются нулями |

В случае некорректного указателя (см. 3.4 Указатели и информация при символьном обмене, стр.11¹) в отклике вернётся exception `02 ILLEGAL DATA ADDRESS` (см. 7 MODBUS Exception Responses, стр. 48²).

## 3.6 Функция `0x03`/`0x7F`(стр.12¹) – запись параметра

### Значение с фиксированной или с плавающей запятой

Реализуется посредством Modbus-функции `0x10 Write Multiple Registers` для записи одного float-параметра (2 регистра) или целочисленного значения – 1 регистр для WORD, 3 регистра для DWORD (старший регистр = `0`) или INT/DINT (старший регистр = `0xFFFF`для отрицательных значений в дополнительном коде, иначе `0x0000`)

Запрос

| Поле | Длина | Значение |
| --- | --- | --- |
| Код функции | 1 байт | `0x10` – Запись регистров хранения |
| Начальный адрес | 2 байта³ | Указатель (= канал × `1000`+ параметр) |
| Количество регистров | 2 байта³ | 2 регистра для REAL, 1 регистр для беззнаковых WORD, 3 регистра для DWORD/DINT |
| Количество записываемых байт | 1 байт | Удвоенное значение предыдущего параметра |
| Данные | – | – |

Отклик при отсутствии ошибок в запросе и успешном выполнении операции совпадает с началом запроса (содержит код функции, начальный адрес, количество регистров)

В случае некорректного указателя (см. 3.4 Указатели и информация при символьном обмене, стр.11¹) в отклике вернётся exception `02 ILLEGAL DATA ADDRESS` (см. 7 MODBUS Exception Responses, стр. 48²).

### ASCII-значение

Реализуется посредством Modbus-функции `0x0F Write Multiple Coils` для записи одного ASCII-значения, количество регистров может быть фиксированным, фактическая длина строки в этом случае будет определена по нуль-терминатору строки

Запрос

| Поле | Длина | Значение |
| --- | --- | --- |
| Код функции | 1 байт | `0x0F` – Запись катушек |
| Начальный адрес | 2 байта³ | Указатель (= канал × `1000` + параметр) |
| Количество катушек | 2 байта³ | Количество записываемых байт × 8 |
| Количество записываемых байт | 1 байт | Определяется записываемым указателем |
| Данные | – | ASCII-строка для записи |

Отклик при отсутствии ошибок в запросе и успешном выполнении операции совпадает с началом запроса (содержит код функции, начальный адрес, количество регистров)

В случае некорректного указателя (см. 3.4 Указатели и информация при символьном обмене, стр. 11¹) в отклике вернётся exception `02 ILLEGAL DATA ADDRESS` (см. 7 MODBUS Exception Responses, стр. 48²).
